# Модалка редактирования компонентов - Спецификация дизайна

## Обзор

Модалка для управления компонентами продукта (конфигуратор бандлов) - дополнительные продукты, которые могут быть опционально или обязательно включены в комплект с основным товаром. Энтерпрайз-уровень сложности с продвинутыми политиками ценообразования.

---

## Добавление компонентов: Продукты vs Варианты

### Концепция

Компонентом может быть:
1. **Простой продукт** - товар без вариантов (например: "Зарядное устройство 65W")
2. **Вариант продукта** - конкретный вариант вариативного товара (например: "Чехол / Чёрный / Кожа")
3. **Вариативный продукт целиком** - все варианты товара как выбор (например: "Чехол" с выбором цвета)

### Два режима добавления вариативного продукта

#### Режим 1: "Отдельные варианты" (по умолчанию)

Каждый выбранный вариант добавляется как отдельная позиция в группу.

**Когда использовать:**
- Разные цены/правила для разных вариантов
- Нужно ограничить доступные варианты
- Каждый вариант - самостоятельная опция

**Результат в админке:**
```
ПОЗИЦИИ (3)
+-------+-------------------------+-------------+----------------+-----------+
| [≡]   | Чехол / Чёрный / Кожа   | ₽4,500      | +10% наценка   | ₽4,950    |
| [≡]   | Чехол / Чёрный / Силикон| ₽2,500      | Без изменений  | ₽2,500    |
| [≡]   | Зарядка 65W             | ₽3,200      | -5% скидка     | ₽3,040    |
+-------+-------------------------+-------------+----------------+-----------+
```

**Результат на витрине:**
```
┌─────────────────────────────────────────────────────────────┐
│ АКСЕССУАРЫ (выберите до 3)                                  │
├─────────────────────────────────────────────────────────────┤
│ [ ] Чехол / Чёрный / Кожа                        ₽4,950     │
│ [ ] Чехол / Чёрный / Силикон                     ₽2,500     │
│ [ ] Зарядка 65W                                  ₽3,040     │
└─────────────────────────────────────────────────────────────┘
```

#### Режим 2: "Продукт с выбором вариантов"

Продукт добавляется как одна позиция, покупатель выбирает вариант на витрине.

**Когда использовать:**
- Одинаковые правила цен для всех вариантов
- Пользователь должен выбрать конфигурацию (цвет, размер)
- Логическая группировка вариантов

**Результат в админке:**

### Обновлённые типы данных

```typescript
// Тип компонента
export enum ComponentItemType {
  SIMPLE_PRODUCT = "SIMPLE_PRODUCT",       // Простой продукт
  SINGLE_VARIANT = "SINGLE_VARIANT",       // Конкретный вариант
  PRODUCT_WITH_VARIANTS = "PRODUCT_WITH_VARIANTS", // Продукт с выбором
}

// Позиция компонента (расширенная)
export interface IComponentItem {
  id: string;
  itemType: ComponentItemType;

  // Для SIMPLE_PRODUCT и PRODUCT_WITH_VARIANTS
  productId: string;
  product?: IProduct;

  // Для SINGLE_VARIANT
  variantId?: string;
  variant?: IProductVariant;

  // Для PRODUCT_WITH_VARIANTS - ограничение доступных вариантов
  availableVariantIds?: string[];        // null = все варианты
  availableOptionValues?: {              // Ограничение по опциям
    optionGroupId: string;
    allowedValueIds: string[];
  }[];
  autoHideOutOfStock?: boolean;          // Скрывать варианты без наличия

  sortIndex: number;

  // Ценообразование (применяется ко всем вариантам)
  priceType: ComponentPriceType;
  priceValue: number | null;

  // Вычисляемые
  basePrice: number;                     // Мин. цена для диапазона
  basePriceMax?: number;                 // Макс. цена (для вариативных)
  finalPrice: number;
  finalPriceMax?: number;

  // Переопределения
  customTitle?: string | null;
  customImage?: IMediaFile | null;

  // Наличие
  isAvailable: boolean;
  stockStatus?: string;
  totalStock?: number;                   // Суммарный сток всех вариантов
}
```

### Сценарии использования

| Сценарий | Рекомендуемый режим | Причина |
|----------|---------------------|---------|
| Чехлы разных цветов с одинаковой ценой | Продукт с выбором | Компактно, один блок выбора |
| Чехлы разных материалов с разной ценой | Отдельные варианты | Цены видны сразу |
| Гарантия 1/2/3 года | Отдельные варианты | Разные услуги, разные цены |
| Подарочная упаковка (3 дизайна) | Продукт с выбором | Одна цена, выбор дизайна |
| Аксессуары из каталога | Отдельные варианты | Разные товары |
| Конфигурация ноутбука (RAM/SSD) | Продукт с выбором | Связанные опции |

### UX-рекомендации

1. **По умолчанию** - режим "Отдельные варианты" (проще понять)
2. **Автоопределение** - если у продукта >4 вариантов, предлагать режим "с выбором"
3. **Превью** - показывать как будет выглядеть на витрине перед сохранением
4. **Валидация** - предупреждать если выбраны варианты без наличия
5. **Быстрые действия** - "Добавить все варианты" / "Убрать все варианты"

## Типы данных (TypeScript)

```typescript
// Типы правил ценообразования
export enum ComponentPriceType {
  BASE = "BASE", // Без изменений, базовая цена продукта
  FIXED = "FIXED", // Переопределение фиксированной ценой
  MARKUP_PERCENT = "MARKUP_PERCENT", // +X% от базы
  DISCOUNT_PERCENT = "DISCOUNT_PERCENT", // -X% от базы
  MARKUP_FIXED = "MARKUP_FIXED", // +X рублей от базы
  DISCOUNT_FIXED = "DISCOUNT_FIXED", // -X рублей от базы
  FREE = "FREE", // 100% скидка, бесплатно в бандле
  INCLUDED = "INCLUDED", // Цена включена в бандл
}

// Режим расчёта бандла
export enum BundleCalcMode {
  ADDITIVE = "ADDITIVE", // Компоненты добавляются к базовой цене
  INCLUSIVE = "INCLUSIVE", // Компоненты включены в базу
  HYBRID = "HYBRID", // Настройка per-group
}

// Группа компонентов
export interface IComponentGroup {
  id: string;
  title: string;
  slug: string;
  sortIndex: number;

  // Правила выбора
  isRequired: boolean;
  isMultiple: boolean;
  minSelection: number;
  maxSelection: number | null;

  // Значения по умолчанию
  defaultItemIds: string[];

  // Режим ценообразования (для HYBRID)
  bundleCalcMode?: BundleCalcMode;

  // Позиции
  items: IComponentItem[];
}

// Позиция компонента (продукт в группе)
export interface IComponentItem {
  id: string;
  productId: string; // Ссылка на вариант/продукт
  product?: IProductVariant; // Разрешённые данные продукта
  sortIndex: number;

  // Ценообразование
  priceType: ComponentPriceType;
  priceValue: number | null; // Для FIXED, MARKUP_*, DISCOUNT_*

  // Вычисляемые (только чтение)
  basePrice: number;
  finalPrice: number;

  // Переопределения
  customTitle?: string | null;
  customImage?: IMediaFile | null;

  // Наличие
  isAvailable: boolean;
  stockStatus?: string;
}

// Шаблон правила ценообразования
export interface IPricingRuleTemplate {
  id: string;
  name: string;
  priceType: ComponentPriceType;
  priceValue: number | null;
  applyToGroupIds: string[] | "all";
}

// Многоуровневая скидка
export interface ITieredDiscount {
  id: string;
  minItems: number;
  discountPercent: number;
}

// Payload модалки
export interface IEditComponentsModalPayload {
  productId: string;
  groups: IComponentGroup[];

  // Глобальные настройки
  bundleCalcMode: BundleCalcMode;
  pricingTemplates: IPricingRuleTemplate[];
  tieredDiscounts: ITieredDiscount[];

  // Настройки отображения
  displayStyle: "accordion" | "tabs" | "flat" | "wizard";
  showImages: boolean;
  showSku: boolean;
  showStock: boolean;
  showComparePrice: boolean;

  // Настройки склада
  outOfStockBehavior: "hide" | "disable" | "backorder";
  inheritStock: boolean;

  // Валидация
  validationMessage: string | null;

  // Колбэки
  onSave: (data: IEditComponentsModalSaveData) => void;
}

export interface IEditComponentsModalSaveData {
  groups: IComponentGroup[];
  bundleCalcMode: BundleCalcMode;
  pricingTemplates: IPricingRuleTemplate[];
  tieredDiscounts: ITieredDiscount[];
  displayStyle: string;
  settings: Record<string, unknown>;
}
```

## Потоки взаимодействия

### Добавление новой группы

1. Клик [+ Добавить группу] в сайдбаре
2. Новая группа появляется внизу с названием "Новая группа"
3. Группа автоматически выбирается, открывается панель деталей
4. Пользователь редактирует название, настройки
5. Пользователь добавляет позиции через [+ Добавить]

### Добавление позиций в группу

1. Клик [+ Добавить] в секции позиций
2. Открывается модалка выбора продуктов (аналогично селектору вариантов)
3. Пользователь ищет/фильтрует продукты
4. Пользователь выбирает один или несколько продуктов
5. Продукты добавляются в группу с правилом BASE по умолчанию

### Редактирование правила цены

1. Клик на ячейку "Правило цены" в AG-Grid
2. Открывается инлайн выпадающий список:
   ```
   +---------------------------+
   | Без изменений (база)      |
   | Фиксированная цена...     |
   | Процентная наценка...     |
   | Процентная скидка...      |
   | Фиксированная наценка...  |
   | Фиксированная скидка...   |
   | Бесплатно (100% скидка)   |
   | Включено в бандл          |
   +---------------------------+
   ```
3. Для опций, требующих значение, появляется инпут
4. Итоговая цена пересчитывается мгновенно

### Сортировка групп

1. Перетаскивание за ручку группы в сайдбаре
2. Бросок на новую позицию
3. Обновление sortIndex

### Сортировка позиций

1. Перетаскивание за ручку строки в AG-Grid
2. Бросок на новую позицию
3. Обновление sortIndex внутри группы
