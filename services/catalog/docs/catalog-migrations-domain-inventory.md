# Catalog Migrations Domain Inventory

Inventory date: 2026-06-30.

This is the accepted final-schema inventory for the handwritten catalog `node-pg-migrate` baseline under `services/catalog/migrations/domains/**`.

Canonical source files:

- `services/catalog/src/repositories/models/index.ts`
- `services/catalog/src/repositories/models/**/*.ts`

DDL conventions used by the baseline:

- `catalogSchema = pgSchema("catalog")` maps to `CREATE SCHEMA IF NOT EXISTS "catalog"`.
- Drizzle `timestamp(..., { withTimezone: true, mode: "string" })` maps to `timestamp with time zone`.
- Drizzle `.defaultNow()` maps to `DEFAULT now()`.
- Drizzle `.defaultRandom()` maps to `DEFAULT gen_random_uuid()`; this inventory explicitly adds `CREATE EXTENSION IF NOT EXISTS "pgcrypto"` in `0002_foundation__extensions.sql`.
- Drizzle `bigint(..., { mode: "number" })` maps to PostgreSQL `bigint`; mode is TypeScript/runtime serialization only.
- Drizzle `jsonb()` maps to `jsonb`; `text().array()` maps to `text[]`; `integer().array()` maps to `integer[]`.
- All foreign keys are `MATCH SIMPLE`, `ON UPDATE NO ACTION`, and `NOT DEFERRABLE` unless the model explicitly represents otherwise. No model currently represents deferrability.
- When Drizzle model code does not provide a constraint name, this baseline chooses stable explicit names: `{table}_pkey`, `{table}_{column}_fk`, `{table}_{columns}_unique`.
- No foreign key is added where the runtime model lacks `.references()` or `foreignKey(...)`, even when a comment implies a relationship.

| `catalog-refs.ts` / `catalogVariant` | Compatibility-only reference object. It is not exported from `models/index.ts`, duplicates the canonical `variant` table partially, and is intentionally not assigned to a separate DDL file. |
| `product-inventory-settings.ts` comment says product FK exists | The model has no `.references()` or `foreignKey(...)`; the baseline does not create that FK. |
| Media file IDs, product/item reference IDs, variant IDs in inventory/cost/bulk tables | No cross-service or implied FK is added unless the runtime model declares it. |
| Deferrability | No runtime model represents `DEFERRABLE` or `INITIALLY`; all constraints use PostgreSQL defaults. |
